# Transport Simulation Game - Design Document

## コンセプト

simutransの戦略的深さを維持しつつ、操作の煩雑さを解消する。

核心: 「現実を模倣するといい結果になる」ゲーム性を守る。

### 残すべき戦略要素

- **フィーダー輸送**: 支線で集めて幹線で運ぶ。路線設計に階層構造が生まれる
- **折り返し運転**: 終端駅での折り返し、途中駅での区間運転
- **需要に応じた複線化**: 単線で始めて、需要増加に合わせて段階的に複線化する判断
- **複数ホームの交互運用**: 高頻度発着を実現するための駅設計の工夫

これらが「知っていれば有利、知らなくても遊べる」深さを生む。

### simutransの問題点

- 1マスずつの敷設が煩雑(長距離路線、複線化)
- 物理レールのグルーピングができず管理が困難
- 自由度の高さが操作コストに直結

## アーキテクチャ

### タイルベース + グラフ操作

- **世界・表示・シミュレーション**: タイルベース
- **操作**: 常にグラフレベル(ノード配置→エッジ接続→タイルに自動展開)
- **1タイル単位の手動操作は提供しない**

グラフが正のデータ。タイル上の線路はグラフから導出される。細かい調整はノードの追加・移動で行う。

### 物理レールと路線の分離

独立した2つの概念:

- **物理レール(グラフ)**: 地形上に敷設されたインフラ。ノードとエッジで構成。共有資源
- **路線**: 物理レール上に定義された運行計画(経路・停車駅・ダイヤ)

1本のレール上に複数の路線が走る。

## グラフモデル

### ノード種別

| 種別 | 容量 | 客扱い | 用途 |
|------|------|--------|------|
| 駅 | ホーム数 | あり | 旅客・貨物の乗降 |
| 信号場 | 線数 | なし | 単線でのすれ違い |
| 信号 | 1 | なし | 閉塞区間の区切り |

### エッジ属性

- **線数**: 単線(同時に1方向のみ) / 複線(双方向同時通行)
- **電化有無**
- **速度プロファイル**: 通過タイルのカーブ・勾配から自動算出

### 駅ノードの詳細

- ホーム数: 同時に停車できる列車数
- ホーム長: 収容可能な編成長を制約
- ホームごとの入場条件: 路線・方向で制限可能

```
例: B駅(3面)
  ホーム1: 東西本線 A方面行きのみ
  ホーム2: 東西本線 C方面行きのみ
  ホーム3: 鉱山支線 専用
```

### 車両

- 編成は長さを持つ
- ホーム長に収まらなければ停車できない
- 閉塞区間の長さにも影響
- 短い編成で頻度を上げるか、長い編成で輸送力を上げるかの判断

### 閉塞

- エッジ内に信号ノードを追加して閉塞区間を細かく設定できる
- 同方向への渋滞(後続列車が前の列車に詰まる)を表現
- 長距離エッジ全体がロックされる問題を回避

### 空間的制約

操作はグラフレベルだが、制約検証はタイルレベルで行う。

- エッジ作成・変更時にタイルに展開し、占有・衝突を検証
- 問題があればフィードバック(「幅が足りない」「立体交差が必要(追加コスト)」等)

### 速度制約

- エッジ内のタイル列から、カーブ・勾配による速度プロファイルを自動算出
- プレイヤーにはエッジの所要時間として表示
- エッジ選択時にボトルネック(急カーブ・急勾配)をハイライト
- 改善手段: ノード追加で経路変更、迂回、トンネル(高コスト)

## 敷設操作

### 基本フロー

1. マップ上にノード(駅)を配置
2. ノード間を接続 → 地形考慮の最適経路が自動計算・プレビュー
3. 確定 → タイルに展開
4. 経路を変えたければ中間にノードを追加

### 部分的な改修

エッジ上の範囲をドラッグ選択 → 複線化等の操作。裏でノード自動挿入・エッジ分割。

```
Before: A ────────────── B (単線)
After:  A ───── C ═════ D ───── B (C-D間だけ複線化)
```

### エッジの管理(区間)

- エッジ(区間)単位で複線化・電化・撤去
- エッジを選択すると使用中の路線一覧が見える
- 改修時に影響する路線を事前に把握できる

## 地形

- 建築・配置・運転を制約する
- 現実的な地形生成
- 経路選択のジレンマを生む:
  - **山岳**: トンネルは高コストだが距離短縮。迂回は安いが時間増
  - **河川**: 橋が必要。幅で架橋コストが変わる
  - **平野**: 安く敷けるが、都市が密集して用地確保が難しい
  - **海岸**: 港湾都市への需要が高いが地形が限られる

## 都市(市域)

- 住居
- 工場(米、歯車、鉄...)
- 需要と供給
- 地形によって適した需要・供給物が配置される
- 加工チェーンによる戦略性:
  - 都市は消費財を受け取ると人口が増える
  - 人口が増えると労働力が発生し、工場の生産力が上がる
  - 工場の産出物を別の都市に運ぶとさらに発展

## プレイヤーの戦略的判断

- どこに駅を置くか(地形・都市・速度のトレードオフ)
- どこを複線にするか
- どこに信号を置いて閉塞を区切るか
- どの駅のホームを増やし、どう割り当てるか
- 路線をどう設計するか(フィーダー構造、直通、区間運転)
- 車両の編成長と頻度のバランス
